# 自建 visual 评估视频清单

如果开源数据不能覆盖某类场景，就按这个清单找/上传视频。

最有价值的是“真实高分辨率源视频”，例如真实 1080p 或 4K。我们会从高分辨率源自动生成 720p/360p 版本，这样同一内容可以公平比较不同分辨率。

## 视频数量建议

第一版建议 8-15 个视频：

| 类型 | 数量 | 分辨率建议 | 目的 |
|---|---:|---|---|
| 体育/动作 | 2-3 | 1080p/4K | 测动作、快速切镜、球/人等小目标 |
| 商品/广告 | 2-3 | 1080p/4K | 测 logo、产品、小物体、边缘信息 |
| 人物/影视/综艺 | 2-3 | 1080p | 测人物主体、多人同框、室内外切换 |
| OCR/文字画面 | 1-2 | 1080p/4K | 测字幕、路牌、屏幕文字 |
| 复杂场景 | 1-2 | 1080p/4K | 测人群、街景、商场、舞台 |
| 低清退化 | 1-2 | 360p/480p/720p | 测低清视频下限 |

## 单个视频要求

优先：

- MP4；
- 30 秒到 5 分钟；
- 有镜头变化，不要全程静态；
- 画面里包含明确目标；
- 最好有原始 1080p/4K；
- 尽量不要严重水印压缩。

可以接受：

- 5-15 分钟长视频，但第一轮评估会截取关键片段；
- 720p 视频，用来测低清/普通素材；
- 手机拍摄视频，只要内容清楚。

暂不优先：

- 只有纯风景、无明确目标的视频；
- 过度模糊/过暗/抖动严重的视频；
- 从 720p 上采样到 4K 的“伪 4K”视频。

## 上传时最好附带的说明

每个视频上传后，最好顺手写几行：

```text
文件名：sports_4k_001.mp4
大概内容：足球比赛，球员射门和观众席
重点时间：
- 00:10-00:18 球员带球
- 00:30-00:35 足球在画面左下角，很小
- 01:02-01:10 右上角有 logo
- 01:20-01:30 字幕/广告牌文字明显
```

不需要一开始特别精确，粗略时间就行。后续我们再转成正式标注。

## 文件命名建议

```text
sports_4k_001.mp4
sports_1080p_002.mp4
ad_product_4k_001.mp4
ad_logo_edge_1080p_001.mp4
movie_people_1080p_001.mp4
ocr_text_4k_001.mp4
street_small_object_1080p_001.mp4
lowres_360p_001.mp4
```

## 必须覆盖的 query 类型

每个视频不需要覆盖所有类型，但整个评估集最好覆盖：

| query_type | 示例 |
|---|---|
| `global_scene` | a football match in a stadium |
| `main_subject` | a woman speaking indoors |
| `small_object` | a phone in someone's hand |
| `edge_object` | logo on the top right |
| `action` | a player kicks the ball |
| `text_overlay` | large text on the screen |
| `style_scene` | night street scene |

## 分辨率对比原则

最公平的对比方式：

```text
真实 4K/1080p 源视频
  ↓
生成 1080p / 720p / 360p 版本
  ↓
用完全相同的 query 和时间段标注评估
```

不要把低清视频上采样成 4K 来证明 4K 效果。上采样只适合测试工程流程，不适合做模型效果结论。
